## 一、技术选型决策

### 最终决策：WinUI3 + C#

| 决策维度 | 选择 | 决策依据 |
|---------|------|----------|
| **UI框架** | WinUI3 | 用户明确需求，微软最新UI规范 |
| **编程语言** | C# | 原生Windows API调用，无桥接层 |
| **平台范围** | Windows Only | 目标用户群体定位 |

**技术选型理由：**
- ✅ 原生支持 Windows Process API（进程监控）
- ✅ 原生支持文件系统操作（存档管理）
- ✅ 原生支持窗口截图API（截图功能）
- ✅ 符合 WinUI3 规范需求
- ✅ 可参考 PotatoVN 开源代码加速开发

### 存档引擎适配说明

存档管理功能的技术实现依赖引擎适配：

| 适配阶段 | 引擎 | 原因 | 预估周期 |
|---------|------|------|---------|
| **第一版** | Renpy | 存档格式简单、开源资料多、覆盖率70% | 2-3周 |
| **第一版** | krkr | Galgame主力引擎、覆盖率90% | 3-4周 |
| **第一版** | Tyrano | 常见Galgame引擎、存档格式相对简单 | 2-3周 |
| **后续** | VNM | 需深入研究存档结构 | 后续规划 |

**引擎检测流程：**
```
游戏扫描 → 文件特征检测 → 引擎识别
    │
    ├── Renpy → 使用 Renpy 存档解析器
    ├── krkr → 使用 krkr 存档解析器
    ├── Tyrano → 使用 Tyrano 存档解析器
    ├── VNM → 显示"暂不支持"提示
    └── 其他 → 显示"请手动指定存档位置"
```

## 三、刮削系统设计

### 刮削源优先级策略

| 优先级 | 来源 | 数据特点 | 用途 |
|-------|------|---------|------|
| 1 | **Bangumi** | 中文信息最全 | 中文名称、简介、标签 |
| 2 | **ymgal** | 中文Galgame站 | 补充中文信息 |
| 3 | **cngal** | 国内资讯站 | 国内游戏资讯 |
| 4 | **VNDB** | 全球最大VN库 | 封面、角色图、英文补充 |

### 刮削匹配策略

| 匹配率 | 处理方式 |
|-------|---------|
| > 90% | 自动采用刮削结果 |
| 70-90% | 弹窗提示，用户确认 |
| < 70% | 用户手动选择刮削源或自定义 |

### 刮削数据优先级

| 数据类型 | 优先来源 | 格式要求 |
|---------|---------|---------|
| 游戏名称 | Bangumi中文 > 原名 | 中文名称优先 |
| 游戏封面 | VNDB/Bangumi | 竖版海报式封面优先 |
| 游戏画板 | VNDB | 横版官方宣传图优先 |
| 游戏简介 | Bangumi | 中文描述优先 |
| 角色/标签 | VNDB + Bangumi | 合并数据 |

## 四、自定义 API 规范

### API 接口概述

| API名称 | 基础路径 | 功能 | 版本规划 |
|---------|---------|------|---------|
| **补丁下载 API** | `/api/v1/patches/*` | 搜索/安装/检测补丁 | 第一版核心 |
| **流程图 API** | `/api/v1/flowcharts/*` | 获取流程图/同步存档位置 | 第一版预留 |
| **成就系统 API** | `/api/v1/achievements/*` | 成就列表/解锁上报/分享卡片 | 第一版预留 |

### 补丁下载 API（第一版核心）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/search` | GET | 搜索匹配补丁 |
| `/install` | POST | 安装补丁 |
| `/status` | GET | 检测补丁安装状态 |

### 流程图 API（预留）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/{game_id}` | GET | 获取游戏流程图结构 |
| `/{game_id}/sync` | POST | 同步存档位置到流程图节点 |

### 成就系统 API（预留）

| 接口 | 方法 | 功能 |
|------|------|------|
| `/{game_id}` | GET | 获取游戏成就列表 |
| `/{game_id}/unlock` | POST | 上报成就解锁 |
| `/share/{achievement_id}` | GET | 生成分享卡片 |

> 详细 API 数据结构见附录。

## 五、风险应对策略

| 风险 | 等级 | 应对策略 |
|------|------|----------|
| **存档引擎适配复杂** | 🔴 高 | 先Renpy+krkr，VNM后续；按库动态检测 |
| **存档替换出错** | 🔴 高 | 三重保障：进程检测→自动备份→用户确认 |
| **流程图API需自建** | 🔴 高 | 第一版预留接口+隐藏功能+后续上线 |
| **刮削匹配不准确** | 🟡 中 | 90%阈值+用户选择+手动编辑 |
| **进程监控影响性能** | 🟡 中 | 高级监控设为可选，默认关闭 |
